# SchatPhone Product Module Feature Catalog / 产品模块功能目录

Updated / 更新时间: 2026-05-04

## 1. 文档定位 / Document Purpose

本文是 SchatPhone 当前所有现有模块的产品功能目录，面向产品经理、设计、运营、测试、后续开发同事、AI 助手，以及不懂编程但需要理解产品的用户。

它不讲代码实现，而是回答：

- 用户从哪里进入这个模块？
- 入口中文名和英文名分别是什么？
- 模块里有哪些一级菜单、二级菜单或主要操作？
- 每个功能在产品体验里解决什么问题？
- 哪些模块是用户前台入口，哪些只是内部协调能力？

相邻文档分工：

- `docs/pm/PRODUCT_MANAGER_PROJECT_BRIEF.md`：项目方向、当前交付范围、PM 级优先级。
- `docs/roadmap/PROJECT_MODULE_AUDIT.md`：模块成熟度、工程风险、候选任务池。
- `docs/product-decisions/FILES_INTERNAL_STORAGE_ROLE.md`：Files 为什么隐藏为内部储存组件。
- 本文：面向非技术读者的“功能字典”和“模块说明书”。

## 2. 阅读方式 / How To Read

| 字段 | 含义 |
| --- | --- |
| 入口名称（中英） | 用户或团队沟通时应使用的模块名称。 |
| Route | 当前项目里的访问路径，供测试、开发、AI 助手定位页面。 |
| 可见性 | 说明该模块是普通前台入口、上下文入口，还是隐藏内部组件。 |

| 次级菜单/操作 | 一级区域里具体可点、可填、可筛选、可执行的功能。 |
| 功能细节（产品视角） | 用非技术语言说明这个功能实际让用户完成什么事。 |

## 4. 手机壳层与导航 / Shell And Navigation

### 4.1 锁屏 / Lock Screen

入口名称：

- 中文：锁屏
- English：Lock Screen
- Route：`/lock`
- 可见性：默认启动页和锁定状态页面。

| 一级菜单/区域 | 次级菜单/操作 | 功能细节（产品视角） |
| --- | --- | --- |
| 锁定状态 | 已锁定、时间样式、锁屏视觉 | 建立“真实手机”开机/锁屏感，让用户先进入手机壳层，而不是直接看到网页应用。 |
| 通知中心 | 通知分组、未读数、最新通知卡片 | 在解锁前展示来自 Chat、Map、Calendar、Gallery、Phone、系统等模块的消息。 |
| 通知跳转 | 点击通知解锁并进入对应页面 | 把提醒变成沉浸式入口，例如点 Chat 通知直接进入对应聊天。 |
| 专注模式 | 每组仅显示最新一条 | 由 More 的 Focus Mode 控制，用于减少重复通知造成的干扰。 |
| 解锁 | 解锁进入主屏 | 从锁屏进入 Home，完成手机使用路径闭环。 |

产品说明：

- 锁屏不是装饰页，它承担“沉浸感起点”和“通知回流入口”。
- 后续视觉重建时应更接近真实手机锁屏，而不是系统后台页。

### 4.2 主屏 / Home

入口名称：

- 中文：主屏 / 桌面
- English：Home
- Route：`/home`
- 可见性：解锁后的主壳层。

| 一级菜单/区域 | 次级菜单/操作 | 功能细节（产品视角） |
| --- | --- | --- |
| 问候区 | 用户姓名、准备就绪提示 | 让手机像“属于用户自己的设备”，而不是公共工具页。 |
| 状态摘要 | 今日节奏、当前主题、主屏页数 | 用轻量信息展示当前手机状态。 |
| 智能面板 | More Labs 只读面板 | 当 More 里 Smart Panel 开关开启时展示，用于验证未来聚合推荐区。 |
| 搜索系统 | Search System 占位入口 | 提供真实手机常见的系统搜索预期，后续可扩展为全局搜索。 |
| Widget | 天气、日历、音乐、系统、快捷爱心、快捷唱片、自定义 Widget | 让主屏有可浏览内容，不只是应用图标列表。 |
| 应用网格 | Network、Wallet、Photos、Themes、Phone、Map、Calendar、Stock、Chat、Contacts、Settings、More | 当前主要前台应用入口。Files 已按产品决策隐藏。 |
| 主屏分页 | 页面圆点 | 表示可有多页桌面，符合手机桌面习惯。 |
| 编辑主屏 | 拖拽、投放、隐藏 Widget、取消选中、完成 | 允许用户整理桌面布局。 |
| 重置布局 | 恢复主屏默认布局 | 当用户布局混乱时可回到默认状态。 |

产品说明：

- Home 是“沉浸手机壳”和“功能模块”的交界面。
- 新模块是否能上 Home，应优先判断它是否会破坏沉浸感或与已有模块归属重复。

### 5.3 网络与 API / Network & API

入口名称：

- 中文：网络与 API / 网络
- English：Network & API / Network
- Route：`/network`
- 可见性：前台主入口，也可从 Settings 进入。

| 一级菜单/区域 | 次级菜单/操作 | 功能细节（产品视角） |

| 供应商模板 | 套用供应商模板 | 帮助用户快速填入常见供应商的接口结构。 |
| API 接口 URL | 自动识别类型 | 根据 URL 判断大致接口类型，例如 OpenAI-Compatible。 |

| 诊断报告中心 | 问题类型、建议处理、供应商、状态码、模型 | 给非技术支持同事提供可转述的故障信息。 |
| 诊断报告中心 | 前往修复、前往推送设置、复制报告、清空 | 将用户引导到对应恢复入口。 |

产品说明：

- Network 是 AI 能力接入中心，但文案应尽量保持“用户配置服务”的感觉。
- 它承担“帮用户知道哪里错了”的职责，而不是只暴露技术错误。

| 关注与持仓摘要 | 关注标的数、持仓标的数 | 快速了解模拟投资列表规模。 |
| 波动提示 | Top movers | 高亮涨跌明显的标的。 |
| 添加模拟标的 | 代码、名称、价格、涨跌幅、持仓数量、备注 | 创建或更新本地模拟标的。 |
| 保存标的 | Save asset | 保存模拟资产；明显波动可进入 Calendar。 |
| 关注与持仓 | 标的卡片 | 查看价格、更新时间、涨跌幅、持仓数量、市值。 |
| 关注与持仓 | 删除 | 移除模拟标的。 |
| Calendar 联动 | 行情复盘线索 | 大幅波动的标的可生成日历复盘提醒。 |

产品说明：

- Stock 是模拟生活信号模块，不是实时行情或真实券商功能。
- 它让用户的手机世界出现“市场波动需要复盘”的日常线索。

## 8. 跨模块归属关系 / Cross-Module Ownership

| 产品对象 | 归属模块 | 消费模块 | 产品规则 |
| --- | --- | --- | --- |
| 用户身份 | Profile | Chat、Settings | Profile 是全局来源；Chat 可做展示级覆盖。 |
| 角色档案 | Contacts | Chat Directory、Chat、Calendar、Wallet context | Contacts 管角色真实资料、素材、知识点。 |
| 聊天会话 | Chat / Chat Directory | Calendar、Wallet、WorldBook | Chat 管消息；Chat Directory 管绑定和会话设置。 |
| 图片/视觉素材 | Gallery | Appearance、Chat、Map、Contacts | Gallery 管素材与删除保护。 |
| 壁纸 | Appearance + Gallery | Home、Lock Screen | Appearance 负责选择；Gallery 提供素材。 |
| 地点/行程/路线 | Map | Calendar、WorldBook、Chat 位置分享 | Map 管移动和地点上下文。 |
| 事件/提醒 | Calendar | Lock Screen、Push、Map/Phone/Stock 来源模块 | Calendar 管确认、时间和推送排程。 |
| 通话记录 | Phone | Calendar | Phone 管通话；未接电话变成 Calendar 线索。 |
| 交易流水 | Wallet | Chat、Contacts | Wallet 管账本；Chat 可生成转账记录。 |
| 模拟标的/行情线索 | Stock | Calendar | Stock 管模拟资产；大波动变成复盘提醒。 |
| 知识点 | WorldBook | Contacts、Chat、Map、Calendar | WorldBook 管可复用上下文。 |

| 真推送订阅 | Settings | Calendar、Map、Chat、Lock Screen | Settings 管推送生命周期；模块消费通知能力。 |
| 文件元数据 | Files | Backup、Diagnostics、未来桥接 | Files 隐藏，不替代任何前台模块归属。 |

## 9. 当前产品优先级提示 / PM Priority Notes

1. 功能代码可继续推进，不必等待完整视觉重建。
2. 视觉重建已独立沉淀在外观相关文档中，当前功能文档不应被视觉风格未定阻塞。
3. Chat、Map、Calendar、Gallery、Contacts、Settings、Network 是当前联动最强的模块组，适合优先做端到端验收。
4. Phone、Wallet、Stock 体量较小，但能给 Calendar、Contacts、Chat 提供生活化连续线索。
5. Files 应继续隐藏，除非未来有明确命名、明确归属、不会破坏沉浸感的新前台形态。

## 10. 建议演示顺序 / Suggested Walkthrough

给非技术同事或用户演示时，建议按以下顺序：

1. Lock Screen -> Home：先建立真实手机沉浸感。
2. Settings -> Network：说明 AI/API 能力如何接入。
3. Profile -> WorldBook -> Contacts：说明用户身份、世界观、角色档案如何准备。
4. Gallery -> Appearance：说明素材如何复用到壁纸和手机外观。
5. Chat Directory -> Chat：说明角色如何进入聊天，以及实际 AI 会话能力。
6. Map -> Calendar：说明行程和地点反馈如何变成提醒。
7. Phone / Stock / Wallet -> Calendar / Contacts / Chat：说明小模块如何提供生活化连续线索。
8. More：说明实验开关和为什么 Files 不作为普通前台 App 展示。

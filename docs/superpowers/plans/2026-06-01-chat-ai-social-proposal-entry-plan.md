# Chat AI Social Proposal Entry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let Chat AI responses submit reviewed role communication-state proposals without directly mutating Chat state.

**Architecture:** Keep the proposal parser in a focused library, keep final state writes inside existing Chat-owned actions, and route AI-generated proposals through `simulationStore.submitChatSocialEventProposal()`. Chat displays assistant messages normally; World Hub remains the high-risk review surface; Contacts remains snapshot-only.

**Tech Stack:** Vue 3, Pinia, Vitest, existing Chat / Simulation stores.

---

## File Structure

- Create `src/lib/chat-ai-social-proposals.js`: normalize optional `socialEvents` from assistant JSON.
- Modify `src/views/ChatView.vue`: prompt the AI for optional proposals and submit parsed proposals after assistant messages are appended.
- Create `tests/chat-ai-social-proposals.test.js`: parser unit coverage.
- Modify `tests/chat-service-ai-reply.test.js` or create `tests/chat-ai-social-proposal-entry.test.js`: end-to-end ChatView coverage with mocked AI.
- Update current handoff docs after code lands.

- [ ] **Step 1: Write parser tests**

Test accepted `socialEvents`, ignored malformed items, and max length trimming:

```js
import { describe, expect, test } from 'vitest'
import { normalizeChatAiSocialEvents } from '../src/lib/chat-ai-social-proposals'

describe('chat AI social proposal normalization', () => {
  test('keeps supported role-side social proposals', () => {
    expect(
      normalizeChatAiSocialEvents({
        socialEvents: [
          { type: 'role_greeting_request', explanation: 'open contact' },
          { eventType: 'role_block_user', reason: 'conflict escalated' },
        ],
      }),
    ).toEqual([
      { eventType: 'role_greeting_request', explanation: 'open contact' },
      { eventType: 'role_block_user', explanation: 'conflict escalated' },
    ])
  })

  test('ignores unsupported, missing, and malformed proposals', () => {
    expect(
      normalizeChatAiSocialEvents({
        socialEvents: [
          null,
          {},
          { type: 'user_block_role', explanation: 'not generated by role' },
          { type: 'role_unblock_user', explanation: 'restored' },
        ],
      }),
    ).toEqual([{ eventType: 'role_unblock_user', explanation: 'restored' }])
  })
})
```

## F. 开发前的稳定性检查（必读）

- 保留结构稳定：`src/router` / `src/stores` / `src/views` / `src/style.css` 不随意改名  
- 主题变量优先：新增颜色/样式尽量放 `src/style.css` 的 CSS 变量里，避免页面硬编码  
- 路由新增流程：先加 view → 再加 router → 再在 Home 入口跳转  
- AI 调用统一：避免组件内直写 `fetch`，统一走 `src/lib/ai.js`  
- 提交前跑三件事：`npm run lint`、`npm run test`、`npm run build`  
- 多设备协作：若换机器/端口修改，回到当前环境前先 `git pull`

# SchatPhone Chat 进度说明

更新时间：2026-02-22

## 1. 当前结论

聊天模块已经从原型进入可用 Beta：
- 底层数据结构稳定
- 手动触发回复链路稳定
- 会话对象分层已经落地（角色/群聊/服务号/公众号）
- 具备继续扩展“模拟社交”能力的基础

## 2. 已完成能力

### 2.1 数据模型

- 会话/消息分离：`conversations` + `messagesByConversation`
- 消息状态：`sending` / `sent` / `failed`
- 草稿、未读、会话排序已打通
- 历史数据自动迁移兼容

### 2.2 调用与错误处理

- 统一 AI 调用入口：`src/lib/ai.js`
- 错误分级提示：URL/鉴权/404/限流/超时/网络/CORS/服务端
- 支持取消当前请求与失败重试

### 2.3 交互与信息架构

- 聊天列表顶部：返回桌面 + 用户状态 + 新建 + 添加服务号
- 用户状态：空闲/忙碌/离开（状态灯）
- 对话页：发送消息后由“触发回复”按钮发起 AI 响应
- 支持连续触发（无新增用户消息也可继续生成）
- 新增会话通讯录：`/chat-contacts`
  - 角色/群聊 与 服务号/公众号分段管理
  - 支持新建、编辑、删除、进入会话
- 服务号/公众号模板在对话页菜单内配置

### 2.4 测试状态

- `tests/chat-store-model.test.js`：通过
- `tests/ai-error-format.test.js`：通过
- 全量测试：通过

## 3. 当前限制

- 自动回复策略尚未做成每会话可配置开关
- 消息长按操作（引用/编辑/重roll）尚未落地
- 调用预算与额度提醒尚未落地
- 会话通讯录缺搜索和批量管理

## 4. 下一步建议（按优先级）

### P0

1. 会话设置页
- 每会话配置：手动/自动回复、回复条数、回复风格、主动开场策略。

2. 消息操作菜单
- 引用、编辑、删除、复制、按轮重roll。

3. 调用预算控制
- 会话调用计数、阈值提醒、触发前确认。

### P1

1. 会话通讯录增强
- 搜索、批量处理、模板预设库。

2. 流式输出（Streaming）
- 助手回复过程可见，状态扩展 `streaming`。

3. 输入体验
- 多行输入、发送键策略、草稿提示强化。

## 5. 验收标准（阶段）

达到以下标准即可进入“聊天增强阶段”：
- 手动触发链路在异常网络下可恢复且不丢会话状态
- 角色会话与服务号会话在设定注入上行为可区分
- 会话对象的新增/编辑/删除不会破坏历史会话展示
- `npm run lint`、`npm run test`、`npm run build` 全通过

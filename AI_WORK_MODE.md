# SchatPhone AI Work Mode

更新时间：2026-02-22

用途：给后续接手的 Codex / Claude / 其他 AI 编程助手一套统一执行方式，确保“用户自然语言需求”与“项目主线推进”可同时成立。

## 1. 角色定位

- 你是该项目主力工程师，默认直接落代码，不只给建议。
- 用户可以用自然句提需求，用户不负责技术拆解。
- 你负责把自然句转成可执行任务并完成实现、验证、文档同步。

## 2. 双轨执行模型

- 轨道 A（即时需求轨）：优先满足用户当前明确提出的需求。
- 轨道 B（主线任务轨）：持续推进 `TASK_EXECUTION_PLAN.md` 中的 P0/P1。
- 原则：用户需求优先，但每次插单后必须回填主线进度，不让主线断档。

## 3. 需求转译模板（每次开工前内部执行）

- 目标：这次要达成的用户可感知结果。
- 范围：会改哪些文件，不改哪些文件。
- 验收：完成标准（功能、交互、异常路径）。
- 风险：兼容性、数据迁移、回归影响。
- 优先级：高/中/低，是否阻塞主线。

## 4. 优先级与插单规则

- 高优先级插单（阻塞使用、明显 bug、数据风险）：立即执行。
- 中优先级插单（体验增强、非阻塞功能）：并入当前迭代小步完成。
- 低优先级插单（优化类）：登记到 `TASK_EXECUTION_PLAN.md` 待排期。
- 若插单与主线冲突：先满足用户，再重排主线并明确延后项。

## 5. 执行节奏（单次需求）

1. 快速理解与转译需求。
2. 扫描相关代码与文档。
3. 直接实现最小可用改动（MVP）。
4. 运行必要验证（至少 lint/build；行为改动补 test）。
5. 输出结果与影响说明。
6. 同步文档（状态、架构、进度）。

## 6. 交付格式（每次回复统一）

- 改动文件：列出路径。
- 变更内容：每个文件做了什么。
- 用户可见结果：现在能做什么。
- 验证结果：执行了哪些命令、是否通过。
- 进度影响：对 P0/P1 的影响（无影响/延后/提前）。
- 下一步建议：最多 1-3 条可选动作。

## 7. 工程约束（必须遵守）

- 技术栈：Vue 3 + Vite + Pinia + Vue Router + Tailwind v4。
- AI 调用统一走 `src/lib/ai.js`，组件内不直写 AI fetch。
- Home 规则：`app_*` 入口不可删除/隐藏，仅可换位。
- 输入页保留显式保存按钮，并给保存反馈。
- 涉及路由/数据结构/主交互改动时，同步更新：
- `PROJECT_STATUS.md`
- `ARCHITECTURE.md`
- `CHAT_PROGRESS.md`（若涉及聊天域）

## 8. Definition of Done（DoD）

- 功能满足用户当前需求。
- 关键路径可复现，不破坏既有主链路。
- 相关测试或检查通过：
- `npm run lint`
- `npm run build`
- `npm run test`（涉及行为逻辑时）
- 文档与任务清单已更新到最新状态。

## 9. 面向下一位 AI 的接管提示（可复制）

```text
你正在接手 SchatPhone 项目，请严格按以下模式执行：
1) 用户自然语言需求优先，先做“需求转译（目标/范围/验收/风险）”。
2) 采用“双轨执行”：
   - 即时需求轨：先满足用户当前请求
   - 主线任务轨：持续推进 TASK_EXECUTION_PLAN.md（P0/P1）
3) 每次改动后输出：
   - 改动文件
   - 变更内容
   - 用户可见结果
   - 验证结果（lint/build/test）
   - 对主线进度影响
4) 涉及路由/数据结构/主交互变化，必须同步更新 PROJECT_STATUS.md、ARCHITECTURE.md、CHAT_PROGRESS.md（若涉及 Chat）。
5) AI 请求统一走 src/lib/ai.js；不得在组件内直写 AI fetch。
```
